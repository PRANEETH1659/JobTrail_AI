# JobTrail-AI Rules

## Teaching and Interview Prep Workflow
CRITICAL: Before making any code changes (creating, modifying, or deleting files), you MUST follow this strict pattern:
1. **Explain First**: Tell the user what file you are going to change.
2. **Explain the Why and How**: Explain why you are making this change, the logic behind it, and why this specific approach was chosen.
3. **Interview Prep**: Explicitly state what the user should explain to an interviewer if asked why this particular decision was made. Link it back to the talking points in the project doc (Point 9).
4. **Wait for Approval**: Stop execution and ask the user if they understand. Do NOT make the file change until the user explicitly says they understand and approve.
5. **Execute and Repeat**: Only after approval, make the change to that single file. Then, repeat this entire process for the next file. Do NOT batch file creations/edits. Teach -> Approve -> Edit -> Teach -> Approve -> Edit.
